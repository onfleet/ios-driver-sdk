Task

An observable model object representing an assigned or active task.

See CompletedTask for completed task model object.

  • Requirements
  • notes

    Notes for the task. Field length should not exceed 10,000 characters.

  • shortId

    A short human readable ID of the task. Use this to refer task’s id in UI.

  • completeAfter

    A date for the earliest time the task should be completed.

  • completeBefore

    A date for the latest time the task should be completed.

  • destination

    Task’s destination.

  • recipients

    List of task’s recipients.

    Note, that displayable recipient’s name can be overriden by overrides. See getNameForPreferredRecipient() to conveniently get printable name of a recipient including overrides.

  • isActive

    Returns true if the task is active (started); false otherwise.

  • pickupTask

    Returns true if the task is for pickup; false for drop off.

  • requirements

    Shows what kind of attachments must be provided prior to completing the task.

  • overrides

    Task overrides.

    Some displayable properties can be overriden by this dictionary. For example recipientName, recipientNotes and recipientSkipSMSNotifications. Please refer to our API docs for more details https://docs.onfleet.com/reference#create-task

  • requestedBarcodes

    List of assigned barcodes some of which may be required and could stop task completion unless they are appended in TaskCompletionDetails when completing a task.

  • isFulfilled
  • organization

    This task’s creator’s organization id

  • executor

    The id of the organization in charge of executing this task (creator delegates to executor)

  • quantity

    Number of items for the task

  • metadata

    Task’s metada. Metadata can be set by an organization and it’s up to the host app how to deal with them.

  • isSelfAssignable

    Boolean which is true if a task can be self-assigned

  • isSelfAssigned

    Boolean which is true if a task has been self-assigned

  • scanOnlyRequiredBarcodes

    if true, only required barcodes can be scanned, additional must be ignored

  • hasRecipientSignature

    if true, recipient signed task remotely and app does not need to send a signature
