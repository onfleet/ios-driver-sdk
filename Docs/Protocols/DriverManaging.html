DriverManaging

Provides API to control everything around Driver and Tasks.

Managing Duty Status

  • isOnDuty

    Returns true if driver is currently on duty; false otherwise.

    This value can be changed any time by driver of platform.

    Changes of this value can be observed using onDuty$ or subscribing to one of following notifications:

    • Notification.Name.driverDidGoOnDuty
    • Notification.Name.driverDidGoOffDuty
    • Notification.Name.driverDidGoOffDutyForced
  • onDuty$

    Subscribe to changes of isOnDuty.

  • setDutyStatus(goOnDuty:completion:)

    Sets driver’s duty status.

    When setting on duty note that this operation will fail if:

    • location permission is not fully granted. See LocationManaging for more info.

    When setting off duty note that this operation will fail if:

    • there is an active task.

    When this operation succeeds Notification.Name.driverDidGoOnDuty or Notification.Name.driverDidGoOffDuty notification is sent.

    Note

    duty status can be changed by platform, for example when dispatcher forces driver off duty. You can observe to duty status changes using notifications or by subscribing to onDuty$ observable. See isOnDuty for more info.

Managing Driver

  • driver

    Returns observable driver model object graph.

    This property is nil until it’s loaded from persistent store or fetched from network. You can subscribe to driverAvailable$ to observe this.

  • driverAvailable$

    Subscribers will be notified when driver property becomes non-nil.

    This can happen when driver object graph is loaded from persistent model or from network.

  • isDriverCommunicationEnabled

    Method to check if an organization allows drivers to initiate communication with the customer

    Returns true if driver can call or text to a customer; false otherwise.

  • uploadProfileImage(imageUrl:progress:completion:)

    Upload driver’s profile photo.

    Please note that changing profile image consists of 3 steps:

    1. calling this method will upload image and return uploadId.
    2. (optional) the uploadId can be used along with DriverManaging.urlForTemporaryProfileImage(for:) to downlaod the image from server if neede to be displayed in the UI.
    3. when the profile si done editing the uploadId must be added to DriverProfileDetails.image and updateProfile(details:completion:) must be called. When change is registered on server Driver.image$ observable property will be changed.
  • urlForTemporaryProfileImage(for:)

    Constructs an url for profile image with provided uploadId. Use this url to download profile image before the profile image is updated.

    This is a convenient way to show profile image in the UI before the profile update is sent to server.

  • updateProfile(details:completion:)

    Updates driver’s profile details.

    Note that only non-nil values will be updated. See DriverProfileDetails for more info.

Starting Tasks

  • activeTask

    An active task. This value will be nil if there is no active task.

    To observe changes to active task subscribe to Driver.activeTask$ observable.

  • start(task:completion:)

    Starts a task (sets as active).

    This operation will fail if:

    • the task is already active
    • driver is off duty

    This operation should show UI warning if:

    • task is started ouside of completion window; see shouldWarnIfTaskStartedBeforeCompleteAfter(for:)
    • task is started out of order and isTaskOrderEnforced returns true.

    When this operation succeeds Notification.Name.driverDidStartTask notification is sent.

  • isTaskOrderEnforced

    If returns true, a task that is being started is not first in list should trigger a warning in UI that it’s out of order.

  • shouldWarnIfTaskStartedBeforeCompleteAfter(for:)

    Determines if UI should show warning upon starting a task if start time is outside of delivery window.

    Note

    This feature must be explicitly enabled by organization. Before a task is started this method should be explicitly checked and if false is returned a warning in UI should be shown. This rule can be enforced, it’s only to warn drivers.

Completing Tasks

  • completedTasks

    Returns an object that provides completed tasks. See CompletedTasksProviding for more info.

  • complete(task:completionDetails:completion:)

    Completes a task.

    An active task must be supplemented with completion details. See TaskCompletionDetails for more info. Some details are required by the task or org’s settings. Before completing a task several requirements and warnings should be checked, read below.

    This operation will fail if:

    • driver is off duty
    • the task is not active
    • task is already completed
    • sosme required completion details are missing:

      See

      see isNotesOnCompletionRequired(for:)

      See

      see isPhotoAttachmentOnCompletionRequired(for:)

      See

      see isSignatureAttachmentOnCompletionRequired(for:)

      See

      see isBarcodeScanningRequired(for:)

    This operation should show UI warning if some recommended completion details are missing:

    See

    see shouldWarnOnOptionalNotesAttachment(for:)

    See

    see shouldWarnOnOptionalPhotoAttachment(for:)

    See

    see shouldWarnOnOptionalSignatureAttachment(for:)

    When this operation succeeds Notificaton.Name.driverDidCompleteTask notification is sent.

Self-Assigning Tasks

  • isSelfAssignmentEnabled

    Returns true if self-assignment is enabled; false otherwise.

    When this feature is enabled by organization driver can self assign (claim) tasks. These tasks are first shared among all drivers in team and can be claimed by individual drivers on first comes first serves bases. When a task is self-assign it will appear in list of tasks under Driver‘s tasks property. To self assign a task see selfAssign(task:completion:).

  • selfAssignableTasks

    A list of tasks that are available for self-assignment.

    To self assign a task see selfAssign(task:completion:).

    Note

    Self-assignable tasks may not always be updated in real time.
  • selfAssign(task:completion:)

    Self-assings a task that is available in selfAssignableTasks.

    This operation will fail if:

    • driver is off duty
    • feature is disabled, see isSelfAssignmentEnabled
    • task is active, see Task.isActive
    • task is not self-assignable, see Task.isSelfAssignable
    • task is already assigned, see Task.isSelfAssigned

    When this operation succeeds Notificaton.Name.driverDidSelfAssignTask notification is sent.
